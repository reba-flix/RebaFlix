# RebaFlix

Production-ready streaming platform scaffold for original movies, TV series, live TV channels, documentaries, trailers, and exclusive content.

## Stack

- Next.js 15, React 19, TypeScript, Tailwind CSS, Framer Motion, Shadcn-style UI primitives
- Next.js API routes, Node.js, Prisma ORM
- PostgreSQL on Supabase
- Supabase Auth with email and Google OAuth
- Cloudflare R2 and Supabase Storage-ready media uploads
- Stripe checkout endpoint and PayPal configuration placeholders
- Vercel deployment target

## Included

- Netflix-style homepage sections and responsive browsing rows
- Movie detail pages with metadata, cast, subtitles, ratings, comments, and related titles
- Series-ready schema for unlimited series, seasons, episodes, progress, and auto-next
- Live TV schema and page with channels, EPG schedules, categories, and HLS playback
- Custom HLS player shell with subtitles, speed, fullscreen, PiP, keyboard shortcuts, cast/AirPlay UI hooks, and resume API support
- User features: auth, favorites, watch later schema, history, ratings, reviews, and notifications
- Search page plus `/api/search` for natural language and future embeddings integration
- Recommendation service seeded by history, genres, trending, and ratings
- Multilingual database support for English, French, Kinyarwanda, Swahili, Arabic, Spanish, dynamic locale flags, and RTL fields
- Admin dashboard shell for analytics, revenue, users, media, channels, uploads, subscriptions, ads, settings, and backups
- Prisma schema, migration SQL, seed data, RLS starter SQL, and deployment notes

## Local Setup

```bash
npm install
cp .env.example .env
npm run db:generate
npm run db:migrate
npm run db:seed
npm run dev
```

Configure Supabase PostgreSQL URLs, Supabase Auth keys, Cloudflare R2, and Stripe before running migrations against a real environment.

## Supabase Auth

Enable Email OTP and Google OAuth in Supabase. Add redirect URLs:

- `http://localhost:3000/auth/callback`
- `https://your-vercel-domain.vercel.app/auth/callback`

Authenticated Supabase users are mirrored into the Prisma `User` table on server requests.

## Storage

Admin uploads go through `/api/uploads` and store objects in Cloudflare R2. Recommended folders:

- `videos/`
- `trailers/`
- `posters/`
- `backdrops/`
- `subtitles/`

For production streaming, store adaptive HLS/DASH manifests generated by Mux, AWS MediaConvert, Cloudflare Stream, or FFmpeg workers.

## Security Checklist

- Enable Supabase Row Level Security with `supabase/rls.sql` as a starting point.
- Keep service-role keys server-side only.
- Put R2 behind a CDN and use signed URLs for premium/private media.
- Add Stripe and PayPal webhook signature verification before activating payments.
- Move rate limiting to Redis/Upstash for multi-instance production.
- Add moderation and report workflows before public comments launch.

## Deployment

1. Create a Vercel project.
2. Add all `.env.example` variables in Vercel.
3. Use Supabase pooled connection for `DATABASE_URL` and direct connection for `DIRECT_URL`.
4. Run migrations through CI or a release job.
5. Deploy with `vercel --prod`.

## Scale Notes

- Keep media on R2/CDN, not the Next.js server.
- Cache catalog APIs with Vercel edge/CDN headers.
- Use Prisma indexes already included for browsing, EPG, history, analytics, and recommendation lookups.
- Use Server Components for catalog pages and client components for playback/auth interactions.
- Add background jobs for recommendations, notifications, transcoding, and analytics rollups.
